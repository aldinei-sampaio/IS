# Proposals

Esta pasta contém propostas de novas funcionalidades, mudanças de arquitetura ou melhorias ainda em discussão.

## Numeração

As proposals são numeradas sequencialmente a partir de `0001`, garantindo ordem cronológica. O número nunca é reutilizado, mesmo que a proposal seja rejeitada ou abandonada.

Use nomes no formato `{número}-{tema}.md` (ex: `0001-autenticacao.md`, `0002-sistema-de-saves.md`).

## Estrutura de uma Proposal

- **Contexto** — por que a mudança é necessária
- **Proposta** — o que se pretende fazer
- **Alternativas consideradas** — outras abordagens avaliadas
- **Impacto** — o que precisaria ser alterado no código existente
- **Status** — `Em discussão`, `Aprovada`, `Rejeitada` ou `Implementada`
